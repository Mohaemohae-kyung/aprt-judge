# APRT Judge/QC v0

APRT Judge/QC is the evaluation module for Automated Progressive Red Teaming.
It receives target responses through observation references, extracts evidence,
evaluates rubric-aligned scores, computes fitness, and routes a final decision.

This repository currently contains the local project skeleton for v0. The design
is intentionally replaceable: judges should be plugin-like evidence producers,
while scoring, fitness, and final decisions remain separate pipeline stages.

## v0 Scope

The intended v0 pipeline is:

```text
RunObservation
-> JudgeService
-> GoalRubricResolver
-> JudgeDispatcher
-> CanaryJudge / RegexJudge
-> EvidenceValidator
-> RubricEvaluator
-> FitnessCalculator
-> DecisionRouter
-> JudgeResult
```

Judge/QC owns:

- target response evaluation
- evidence creation, validation, normalization, and deduplication
- rubric-based raw scores
- fitness calculation
- decision routing
- JudgeResult creation
- calibration and reward-hacking review documentation

Judge/QC does not own:

- attack generation or mutation operators
- target execution or harness implementation
- archive add/sample behavior
- descriptor classification
- evolution loop or champion selection
- full TraceCheckJudge implementation

Shared APRT objects such as AttackGenome, RunObservation, and ArchiveEntry
should be represented here only through minimal external reference types.

## Design Constraints

- Judges generate Evidence; they do not compute final fitness.
- Fitness is computed by FitnessCalculator.
- Decisions are generated by DecisionRouter.
- Thresholds, decision caps, and enabled judges should remain configurable.
- CanaryJudge must not be frozen as a simple exact-match final verdict.
- Evidence must not store plaintext secrets or raw canary values.
- Attack, Harness, and Archive responsibilities should not be pulled into this module.

## Current State

This skeleton intentionally does not implement CanaryJudge, RegexJudge, schemas,
fitness calculation, decision routing, classifier judging, LLM judging, or
TraceCheckJudge behavior yet.
